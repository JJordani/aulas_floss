\begin{frame}
	\frametitle{Definição de Software de Código Aberto}
	% Fonte: Open Source Initiative, http://www.opensource.org/docs/osd
	% Licença: Creative Commons Attribution 2.5 License
	Toda licença de distribuição de OSS deve respeitar os critérios:
	{\small 
    \begin{itemize}
\item[1] Redistribuição livre
\item[2] Código fonte incluído ou acessível
\item[3] Trabalhos derivados são permitidos\pause
\item[4] Integridade do código fonte do autor, restrições controladas à modificação (via \textit{patches})
%The license may restrict source-code from being distributed in modified form only if the license allows the distribution of "patch files" with the source code for the purpose of modifying the program at build time. The license must explicitly permit distribution of software built from modified source code. The license may require derived works to carry a different name or version number from the original software.
\item[5] Sem discriminação contra pessoas ou grupos
\item[6] Sem discriminação contra áreas do conhecimento ou empreendimentos
%The license must not restrict anyone from making use of the program in a specific field of endeavor. For example, it may not restrict the program from being used in a business, or from being used for genetic research.
% A cláusula acima parece entrar em conflito com a GPL v3 na questão da Tivoization.

%\item[7] Distribuição da licença
%The rights attached to the program must apply to all to whom the program is redistributed without the need for execution of an additional license by those parties.
\item[8 e 9] A licença não pode ser específica para um produto ou distribuição e não deve restringir outros programas
%The rights attached to the program must not depend on the program's being part of a particular software distribution. If the program is extracted from that distribution and used or distributed within the terms of the program's license, all parties to whom the program is redistributed should have the same rights as those that are granted in conjunction with the original software distribution.

%No item 9, "outros programas" não é a mesma coisa que outras partes de um programa ao qual o programa original, modificado ou não, foi anexado. A idéia é impedir que a licença de um programa que é distribuído "empacotado" com outros influencie as licenças desses outros. Isso poderia ser usado para impor restrições de uso de forma colateral. 
%The license must not place restrictions on other software that is distributed along with the licensed software. For example, the license must not insist that all other programs distributed on the same medium must be open-source software.

%\item[10] A licença deve ser neutra, em se tratando de tecnologia
    \end{itemize}}

% rodar um mesmo programa em máquinas de fabricantes distintos e tornou-se mais fácil
% construir programas mais complexos. Neste cenário, três fatores levaram os fabricantes
% de hardware a se preocupar com software: (i) a emergente indústria de software; 
% (ii) leasing de máquinas, que funcionava como um aluguel e não trazia nenhum lucro
% em termos de software; e (iii) a capacidade de vários clientes dos fabricantes de
% hardware de atender suas próprias demandas, em termos de software. Software cada vez 
% mais tornava-se um diferencial competitivo e simplesmente deixar qualquer um 
% beneficiar-se disso parecia uma má estratégia de negócios. Competidores poderiam tomar
% como base o software produzido por uma empresa, melhorá-lo e vendê-lo como seu próprio
% produto. Fabricantes passaram então a dificultar a obtenção do código fonte dos programas
% e a exigir que a lei de copyright se fizesse cumprir. Na segunda metade da
% década de 70, popularizou-se a prática de vender licenças de uso, como as que
